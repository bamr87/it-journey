# Web Development — Section Guide

> Voice/values live in `_data/brand/`. State only web-dev-specific deltas here.

## Audience

Learners building their first sites and practitioners shipping front-ends —
Jekyll, React, GitHub Pages, static-site patterns.

## Voice profile

`practitioner-chronicle` (see `../voice.md`). **Section delta:** show the rendered
result, not just the code — a screenshot/diagram placeholder where UI changes.

## Tone

Build-along and encouraging. Meet beginners where they are (this is a common entry
point for `n00b` → her0), but don't talk down to anyone.

## Structure

Inherits the `posts.instructions.md` skeleton. "What You'll Build" + a concrete
deliverable; platform notes when setup differs; verify by loading the page.

## Do / Don't

- **Do:** prefer the open/free path (GitHub Pages, OSAP); show the result;
  cross-link the relevant quest.
- **Don't:** "seamless integration" / "powerful framework" filler; skip the build
  step that actually breaks for beginners.

## Example openings

- "GitHub Pages is the hidden gem of free hosting — once you get past the one
  config that trips everyone up."
- "Adding an AI chatbot to a Jekyll site sounds heavy. It's about 40 lines. Here's
  every one of them."
