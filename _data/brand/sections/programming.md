# Programming — Section Guide

> Voice/values live in `_data/brand/`. State only programming-specific deltas here.

## Audience

Learners leveling up their craft and practitioners after a cleaner pattern —
refactoring, modular architecture, debugging real failures.

## Voice profile

`practitioner-chronicle` (see `../voice.md`). **Section delta:** show the code, the
diff, and the reasoning; favor the readable solution over the clever one (KIS).

## Tone

Craft-focused and pragmatic. Code teaches — annotate *why*, not just *what*. Own the
bug that caused the post (DFF).

## Structure

Inherits the `posts.instructions.md` skeleton. Every code block has a language tag;
show before/after for refactors; include a verify/expected-output where you can.

## Do / Don't

- **Do:** tailor examples to a real problem; explain trade-offs; link the language
  doc once (DRY).
- **Don't:** generic stock snippets; "powerful framework" filler; clever one-liners
  presented without explanation.

## Example openings

- "The KeyError only showed up in CI. The fix took one line — finding it took the
  rest of this post."
- "Ten shell scripts, three copies of the same function. Here's how I made it one."
