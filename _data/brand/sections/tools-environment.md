# Tools & Environment — Section Guide

> Voice/values live in `_data/brand/`. State only tooling-specific deltas here.

## Audience

Practitioners and contributors tuning their setup — editors, terminals, CLIs,
local toolchains — who want a config that works and saves time.

## Voice profile

`practitioner-chronicle` (see `../voice.md`). **Section delta:** show the actual
config/dotfile and what it changed; prefer open tools (OSAP) and the simplest setup
that works (PoLR/KISS).

## Tone

Hands-on and time-conscious. The payoff is time saved or friction removed — name it.

## Structure

Inherits the `posts.instructions.md` skeleton. Show the before/after of the config;
include install/verify steps; note cross-platform deltas.

## Do / Don't

- **Do:** paste the real config; flag paid vs free tools; show the verify.
- **Don't:** "powerful/seamless tool" filler; tool tourism with no working setup;
  hide that something requires a paid tier.

## Example openings

- "My terminal startup was 1.2 seconds of noise. Here's the config that made it
  instant."
- "VS Code + GitHub Pages can be a full CMS. Here's the exact setup I run."

`cli`, `dotfiles`, `git`, `productivity`, `configuration`.
